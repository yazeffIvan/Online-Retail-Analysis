# Анализ данных интернет-магазина (Online Retail)

## Описание проекта
**Цель:** Провести анализ клиентской базы и транзакций интернет-магазина для выявления ключевых сегментов и разработки маркетинговых рекомендаций.

**Заказчик:** Отдел маркетинга.
**Бизнес-вопрос:** Как эффективно распределить бюджет на удержание клиентов?

## Используемые инструменты
- **Язык:** Python (Pandas, NumPy, Matplotlib, Seaborn)
- **База данных:** SQL Server (аналитические запросы)
- **Среда разработки:** PyCharm

## Файлы в репозитории
- `analysis.py`: Основной скрипт для загрузки, очистки, анализа и визуализации данных.
- `Scripts.sql`: Набор SQL-запросов для расчета ключевых метрик.
- `clean_online_retail.csv`: Очищенный датасет.
- `requirements.txt`: Список зависимостей Python для воспроизведения окружения.

## Ключевые результаты
*   **Общая выручка:** 8.83 млн у.е.
*   **Ключевой инсайт:** 80% выручки приносят 20% клиентов (сегменты Tier 1 и Tier 2).
*   **RFM-сегментация и стратегии:**

| Сегмент | Доля клиентов | Рекомендуемая стратегия |
| :--- | :--- | :--- |
| **Tier 1 (Чемпионы)** | 11.9% | VIP-программа, ранний доступ к новинкам. |
| **Tier 2 (Лояльные)** | 8.6% | Персональные рекомендации, кросс-продажи. |
| **Tier 3 (Спящие)** | 30.4% | Реактивация через промо-код. |
| **Tier 4 (Потерянные)** | 49.2% | Не тратить бюджет. |
